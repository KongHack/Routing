# Route Attribute (`#[Route]`)

## Purpose

The `#[GCWorld\Routing\Attributes\Route]` attribute provides a modern, PHP 8 native way to define routes directly on your handler classes. This approach enhances code readability and organization by co-locating route definitions with the classes responsible for handling them.

## Usage

You apply the `#[Route]` attribute directly above the class definition of your handler. A class can have multiple `#[Route]` attributes if it handles multiple route patterns.

```php
<?php

namespace App\Handlers;

use GCWorld\Routing\Attributes\Route;
use GCWorld\Routing\Attributes\RouteMeta;
use GCWorld\Routing\Attributes\RoutePexCheck;
use GCWorld\Routing\Enumerations\RoutePexCheckType;

#[Route(
    patterns: ['/user/:id', '/profile/:id'], // Defines the URL patterns this class handles
    name: 'user_profile',                    // Unique name for this route, used for reverse routing
    session: true,                           // Indicates if a session should be started for this route
    autoWrapper: true,                       // If a page wrapper should be automatically applied
    title: 'User Profile',                   // Default title for pages generated by this handler
    meta: [
        new RouteMeta(key: 'description', value: 'View and edit user profile.'),
        new RouteMeta(key: 'keywords', value: 'user, profile, account'),
    ],
    pex: [
        new RoutePexCheck(type: RoutePexCheckType::STANDARD, pexString: 'USER_CAN_VIEW_PROFILE'),
        new RoutePexCheck(type: RoutePexCheckType::MAX, pexString: 'USER_MAX_LEVEL_5'),
    ],
    preArgs: ['fixed_prefix_arg'],
    postArgs: ['fixed_suffix_arg']
)]
class UserHandler 
{
    public function get(int $id)
    {
        // Logic to handle GET /user/{id} or /profile/{id}
        // $id will be automatically passed from the URL
    }

    // Other methods like post(), deleteXHR(), etc.
}

```

## Constructor Parameters

The `#[Route]` attribute's constructor accepts the following parameters:

*   **`patterns: array`** (Default: `[]`)
    *   An array of string patterns that define the URLs this route will match.
    *   Placeholders like `/:id/` or `/:slug/` can be used. These are converted into regular expressions during processing using `ROUTING_TOKENS` (see `RouteDiscovery.php` documentation).
    *   Example: `['/blog/:slug', '/article/:slug']`

*   **`name: string`** (Default: `''`)
    *   A unique name for the route. This name is crucial for generating URLs using reverse routing (`CoreRouter::reverse()`).
    *   The processor will append `_X` to the name, where X is the number of parameters in the pattern, to ensure uniqueness if multiple patterns with the same base name exist but differ in parameter count.
    *   Example: `'blog_post_view'`

*   **`autoWrapper: bool`** (Default: `false`)
    *   If `true`, and a `PageWrapper` is configured in `CoreRouter`, the output of the handler method will be automatically wrapped. This is useful for applying consistent page layouts.

*   **`session: bool`** (Default: `false`)
    *   If `true`, `CoreRouter` will ensure a PHP session is started before executing the handler method and performing any permission checks.

*   **`title: string`** (Default: `''`)
    *   A default title for the page. Handler methods can override this. Useful if `autoWrapper` is enabled.

*   **`meta: array`** (Default: `[]`)
    *   An array of `#[GCWorld\Routing\Attributes\RouteMeta]` attributes.
    *   `#[RouteMeta(key: string, value: string)]`: This attribute allows you to define arbitrary key-value metadata associated with the route, which can be used by your application (e.g., for SEO, breadcrumbs, or custom rendering logic).
    *   Example: `[new RouteMeta(key: 'layout', value: 'two-column')]`

*   **`preArgs: array`** (Default: `[]`)
    *   An array of strings. These arguments will be prepended to the list of arguments passed to your handler method, *before* any arguments derived from the URL pattern.
    *   Example: `['some_service_id', 'config_flag_true']`

*   **`postArgs: array`** (Default: `[]`)
    *   An array of strings. These arguments will be appended to the list of arguments passed to your handler method, *after* any arguments derived from the URL pattern.

*   **`pex: array`** (Default: `[]`)
    *   An array of `#[GCWorld\Routing\Attributes\RoutePexCheck]` attributes. This is used for permission checking if a PEX-compatible user class is configured in `CoreRouter`.
    *   `#[RoutePexCheck(type: RoutePexCheckType, pexString: string)]`: Defines a permission check.
        *   `type`: An enum `GCWorld\Routing\Enumerations\RoutePexCheckType` specifying the type of check:
            *   `RoutePexCheckType::STANDARD`: Standard permission check.
            *   `RoutePexCheckType::ANY`: Check if any of the provided permissions match.
            *   `RoutePexCheckType::EXACT`: Check for an exact permission match.
            *   `RoutePexCheckType::MAX`: Check against a maximum permission level.
        *   `pexString`: The string representing the permission to check (e.g., `'CAN_EDIT_ARTICLE'`, `'USER_LEVEL_[2]'` where `[2]` could be a placeholder for a URL parameter).
    *   Example: `[new RoutePexCheck(type: RoutePexCheckType::STANDARD, pexString: 'module_access_user')`

## Internal Method: `getRouteArray()`

*   **`getRouteArray(): array`**:
    *   This public method is primarily used internally by the `Processor`. It converts the attribute's properties into a structured array format that the `Processor` can use to generate the optimized route files. Application developers typically do not need to call this method directly.
